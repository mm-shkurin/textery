# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — [SemVer](https://semver.org/lang/ru/).

Зачем файл нужен именно здесь: `frontend/` публикуется в двух формах — как каталог монорепозитория
и как корень отдельного репозитория (gitverse `slide_frontend`). Версия в `package.json` — это
единственное, по чему собранный артефакт можно сопоставить с кодом; до 0.1.0 она была `0.0.0`, то
есть не сопоставлялся никак.

## [Unreleased]

### Исправлено

- Поллинг статуса генерации больше не накладывает запросы друг на друга: интервал 5с при таймауте
  запроса 25с приводил к нескольким одновременным проверкам, и каждая тратила попытку из бюджета
  `MAX_POLL_ATTEMPTS` — «~5 минут» могли истечь за считаные секунды.
- Истёкший по таймауту запрос теперь ещё и прерывается (`AbortController`), а не только перестаёт
  ожидаться. Таймер по-прежнему отклоняет промис сам — соединение, которое не уважает abort, было
  и остаётся причиной, по которой одного сигнала недостаточно.

### Добавлено

- `npm run audit` — проверка production-зависимостей (уровень `high` и выше) в обоих пайплайнах.
- Пофайловый пол покрытия (`scripts/check-per-file-coverage.mjs`): общий процент не ловит модуль,
  который не тестирует никто.
- Запрет `dangerouslySetInnerHTML` (`react/no-danger`): токены лежат в `sessionStorage`, и
  единственное, что делает это приемлемым, — отсутствие внедрения неэкранированной разметки.

## [0.1.0] — 2026-07-24

Первая пронумерованная версия. Ниже — не полная история (она в `git log`), а состояние среза.

### Добавлено

- Посадочная страница, регистрация с подтверждением кода, вход (в том числе OAuth VK/Yandex),
  генерация документа с поллингом статуса, ручной редактор на Tiptap, история работ
  с keyset-пагинацией.
- `ErrorBoundary` на корне приложения и вокруг редактора: падение поддерева показывает сообщение
  вместо белого экрана.
- `npm run ci:parity` — сравнивает набор npm-скриптов в двух пайплайнах CI. Раньше они
  синхронизировались руками, и расхождение никак себя не проявляло.

### Изменено

- Декоративные изображения героя переведены из PNG в WebP: ~6.5 МБ → ~0.8 МБ на первом экране.
- `<html lang>` — `ru` вместо `en`: весь текст приложения русский, а от `lang` зависит
  произношение в скринридере.
- Логика отправки формы входа вынесена из `LoginForm` в `hooks/useLoginSubmit`.
