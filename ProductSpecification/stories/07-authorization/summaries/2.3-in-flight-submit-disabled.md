## red-selenium (2026-07-14)

**Mistake:** First `_count_requests_to` implementation counted all CDP `Network.requestWillBeSent` events matching the URL substring, with no HTTP method filter.
**Why wrong:** CORS preflight `OPTIONS` requests hit the same URL as the real `POST`, making the duplicate-request assertion permanently unsatisfiable even for correct production code.
**Correct location/approach:** Filter to `method == "POST"` in `base_frontend_statements._count_requests_to`; also added a settle-sleep before reading the log since CDP delivery is async relative to the triggering click.

**Quirk:** A genuinely disabled HTML button blocks even a raw JS `.click()` call — the browser skips activation behavior entirely, so no click event reaches any handler.
**Where:** `register_page_statements.click_submit_button_ignoring_disabled_state` / `base_frontend_statements._assert_disabled`.
**Implication:** Once a button is confirmed disabled, a "second click doesn't duplicate the request" assertion adds no independent signal — it will always pass as a side effect of the disabled check, not because app code actually guarded against a real double-submit.
