## red-frontend (2026-07-14)

**Mistake:** LoginForm.tsx initially re-implemented the placeholder in-flight boundary (isSubmitting state, 500ms setTimeout, try/finally) inline instead of reusing the `useSubmitPlaceholder` hook that RegisterForm.tsx already used for the identical pattern.
**Why wrong:** two independent copies of the same TDD-placeholder logic drift once the real API lands — a fix to one (unmount guard, error handling, abort logic) wouldn't propagate to the other.
**Correct location/approach:** extracted `frontend/src/features/auth/hooks/useSubmitPlaceholder.ts`; both LoginForm and RegisterForm now consume it.

**Quirk:** A genuinely disabled button only blocks that control — a form's native Enter-key submit from a focused text input bypasses the submit button's `disabled` attribute entirely and can still fire the submit handler.
**Where:** `useSubmitPlaceholder.ts` (shared by LoginForm/RegisterForm), any future confirm-button wiring on VerifyCodeForm.
**Implication:** DOM-level `disabled` alone is not a sufficient re-entry guard; `submitWithPlaceholderDelay` now has an explicit `if (isSubmitting) return` early-return guard before setting state.
