## align-design (2026-07-14)

**Mistake:** First align-design pass used CSS `order` on `.verify-resend button`/`[data-testid="verify-resend-countdown"]` to visually match the mockup's countdown-above-button layout without touching JSX.
**Why wrong:** `order` only changes visual position, not DOM/reading order — screen-reader and tab order stayed button-then-countdown, diverging from what's seen (WCAG 1.3.2 meaningful-sequence mismatch).
**Correct location/approach:** Swap the JSX element order directly (countdown span before button in `VerifyCodeForm.tsx`) and drop `order` from the CSS entirely.

## green-selenium (2026-07-14)

**Decision:** Host-side `BACKEND_PORT` moved from 8000 to 8001 in `infra/.env`.
**Why:** Port 8000 was already bound by an unrelated container (`random_coffee_app`) from a different project on the same host; nginx proxies to `backend:8000` on the internal Docker network regardless of the host-side mapping, so the remap is transparent to the frontend.
**Where applied:** `infra/.env` (`BACKEND_PORT`).

## red-frontend (routing) (2026-07-14)

**Quirk:** Dispatching `agent-review`/`premortem` over a commit range that starts right after the commit which actually added the new test (e.g. `8fc3e1e..f9d594d` where `8fc3e1e` added the test and `f9d594d` only advanced progress.md) makes both passes falsely BLOCK, believing the red-phase test was never committed.
**Where:** `/continue`'s pre-commit review dispatch (review-pass range selection).
**Implication:** When a red-phase test lands via an intermediate commit (e.g. a test-review commit) before the behavior commit, widen the review range to include that intermediate commit, not just the final behavior commit.
