# Interview — Story 5: Manual input mode (non-AI document creation)

## Scope

Пользователь выбирает тип документа (доклад/эссе/сочинение/реферат — те же четыре
типа, что и в генерации), но вместо запуска ИИ-генерации сразу попадает в пустой
документ и пишет текст сам, вручную, в редакторе.

В скоуп стори 5:

- Выбор типа документа (переиспользуем `document_type` из story 1).
- Создание пустого документа (без вызова LLM, без полей тема/объём/требования).
- Базовое форматирование в редакторе: заголовки, абзацы, списки, жирный, курсив.
- Сохранение документа.

Вне скоупа (явно отложено):

- Preset/шаблон со структурой по типу документа (готовые заголовки-плейсхолдеры типа
  "Введение / Основная часть / Заключение" для доклада). ACTION: зафиксировать как
  идею/технический долг на будущее — не в стори 5.
- Продвинутое форматирование редактора (автосохранение, экспорт, таблицы, стили) —
  story #10 (Text Editor polish). Стори 5 делает только базовый набор форматирования,
  необходимый чтобы им можно было пользоваться.
- Редактирование уже сгенерированного ИИ текста, работа над "своим" уже написанным
  текстом с последующей ИИ-доработкой — DECISION: это отдельное, более позднее
  направление (объединение ручного и ИИ-режима на одном документе), явно не в этой
  стори. Стори 5 — только "с нуля, руками, без ИИ".
- Auth/user_id — как и story 1, стори 5 анонимная (зависимость на story #7 не блокирует).

## Key Architectural Decisions

- DECISION: документ создаётся пустым, без preset-структуры. Идея с шаблонами
  (заголовки-заготовки по типу документа) зафиксирована как долг/будущая доработка,
  не блокирует эту стори.
- DECISION: базовое форматирование (заголовки/абзацы/списки/жирный/курсив) — в
  скоупе стори 5, поскольку без него редактор нельзя использовать по назначению.
  Продвинутый polish (автосохранение, экспорт, таблицы) — story #10.
- DECISION: связь с ИИ-режимом (доработка сгенерированного текста, доработка своего
  текста через ИИ) — будущее направление, не часть стори 5.

## Business Rules & Constraints

- `document_type` обязателен при создании (те же 4 значения, что в story 1).
- Пустой документ создаётся сразу после выбора типа, без промежуточной формы
  (нет полей тема/объём/требования — в этом ключевое отличие от ИИ-режима).

## Already Implemented (REUSE)

- `document_type` enum/value object — переиспользуется из story 1's `Generation`/
  `Document` shape (frontend: тип фиксирован уже как отдельное поле в story 1's
  generation flow).

## NOT Yet Implemented (Gaps)

- Экран выбора режима (ИИ-генерация vs ручной ввод) на этапе выбора типа документа.
- Пустой документ + редактор с базовым форматированием (заголовки/абзацы/списки/
  жирный/курсив) для ручного режима.
- Сохранение вручную написанного документа.

## Cross-Story Dependencies

- Независима от порядка с #2–#4, #6, #8 (см. stories.md: "#5–#8 may interleave").
- Переиспользует `Document`-модель story 1, но не зависит от завершения генерации —
  создаёт документ напрямую, без `Generation`.
- Пересекается с будущим направлением "ИИ + ручной режим на одном документе" — не
  блокирует стори 5, но стори 5 не должна закрывать дверь для этого (не хардкодить
  "ручной документ" как несовместимый с последующей ИИ-доработкой на уровне модели).
- Базовое форматирование здесь, полный polish — story #10 (Text Editor polish).

## Testing Considerations

- Базовое форматирование тестируется в стори 5 (заголовки/абзацы/списки/жирный/
  курсив); автосохранение/экспорт/расширенные стили — тесты story #10, не здесь.
