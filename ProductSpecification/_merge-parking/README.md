# Merge parking — незакрытое расхождение спеки между слоями

Это **не** папка с документацией. Это отложенный конфликт, и он ждёт решения человека.

## Что произошло

Ветки `dev` (бэкенд story 5) и `feature/05-manual-mode-frontend` независимо написали одни и
те же файлы спецификации. При слиянии git показал 12 конфликтов, из них **ни одного в коде** —
только спека и `acceptance/conftest.py`.

Решить их «на глаз» значило бы объявить, чья версия API-спеки авторитетнее — а это ровно то,
что `CLAUDE.md` запрещает: shared-файлы правятся только в секции своего слоя, «never rewrite
the other layer's rows». Поэтому слияние прошло так:

- на **канонических путях** осталась версия из `dev` (её писала бэкенд-сессия);
- версия из фронтенд-ветки лежит здесь, в `frontend-branch/`, с сохранением относительных путей;
- ничего не потеряно — обе версии в истории и на диске.

## Что здесь лежит

`frontend-branch/` — 11 файлов из `feature/05-manual-mode-frontend`:

| Файл | Природа расхождения |
|------|---------------------|
| `api-specs/documents_{create,get,save}.yaml` | обе ветки создали спеку с нуля (add/add) |
| `stories/05-manual-mode/{05_ManualMode,functionality,interview}.md` | то же |
| `stories/05-manual-mode/tests/{01_API_Tests,05_Security_Tests}.md` | то же |
| `stories/07-authorization/{carryover,progress}.md` | то же |
| `stories.md` | ролл-ап: см. ниже, отдельный случай |

## `stories.md` — почему его нельзя слить механически

Колонки принадлежат разным слоям: `Back`/`Intg`/`Sec`/`Load`/`Infra` — бэкенду, `Front` — нам.
Их можно было бы взять union'ом. Но `Tests` и `%` — это **ролл-ап по обоим слоям**, и стороны
насчитали:

- `dev`: story 5 → `0/40` (0%), story 7 → `26/63` (41%)
- фронтенд-ветка: story 5 → `19/20` (95%), story 7 → `13/24` (54%)

Каждая ветка считала только свои сценарии. Итоговое число не знает ни одна из них — оно
требует обоих списков сразу. Любая цифра, которую я бы вписал, была бы выдуманной, поэтому
на канонический путь легла версия `dev` как есть: она занижает прогресс фронтенда, но не врёт
о том, чего никто не проверял.

## Что НЕ припарковано (и почему)

`acceptance/conftest.py` разрешён сразу, union'ом, и это не суждение о вкусе:

```python
from statements.auth_statements import AuthStatements            # dev
from statements.verify_statements import VerifyStatements        # dev
from statements.frontend.auth.login_page_statements import ...   # фронтенд-ветка
```

Конфликтовал только блок импортов. Фикстуры **обеих** сторон слились чисто и уже ссылаются на
оба набора имён, а оба набора файлов существуют после мержа. Взять одну сторону — получить
`NameError` на фикстурах другой. Union — единственная резолюция, при которой файл вообще
работает.

## Как это закрыть

Нужны обе сессии (или тот, кто владеет спекой):

1. По каждому файлу в `frontend-branch/` — сравнить с каноническим путём, решить, что верно.
2. `stories.md` — свести `Tests`/`%` по фактическим спискам сценариев обоих слоёв.
3. Удалить `ProductSpecification/_merge-parking/` целиком.

Пока папка существует — расхождение открыто. Её присутствие и есть тикет.
